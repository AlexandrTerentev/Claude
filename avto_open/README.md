# AvtoOpen

Кроссплатформенный лаунчер сборок с поддержкой автоматизации для Windows и Linux.

## Возможности

- Создание "сборок" - наборов приложений с настроенными действиями
- Параллельный запуск нескольких приложений одновременно
- Автоматизация через эмуляцию ввода (клики мыши, ввод текста, горячие клавиши)
- Локальное хранение конфигурации в JSON формате
- Простой и понятный графический интерфейс

## Установка

```bash
# Клонирование репозитория
git clone <repo-url>
cd avto_open

# Установка зависимостей
pip install -r requirements.txt

# Для Linux может потребоваться установка дополнительных пакетов:
# sudo apt install python3-tk python3-dev
```

## Запуск

```bash
python src/main.py
```

## Использование

1. **Создать сборку** - нажмите кнопку "Создать" и введите название сборки
2. **Добавить приложения** - в редакторе сборки нажмите "Добавить приложение" и выберите исполняемый файл
3. **Настроить действия** - дважды кликните на приложение для редактирования автоматических действий:
   - **Задержка** - пауза перед следующим действием
   - **Клик мыши** - клик в указанных координатах (есть помощник для захвата координат)
   - **Ввод текста** - автоматический ввод текста
   - **Горячие клавиши** - нажатие комбинаций клавиш (например: `ctrl+shift+s`)
   - **Прокрутка** - прокрутка колесом мыши
4. **Запустить сборку** - выберите сборку из списка и нажмите "Запустить"

## Пример использования

Создание сборки для стриминга:
1. **Discord** → задержка 3 сек → клик по окну чата → ввод сообщения
2. **OBS Studio** → задержка 2 сек → горячая клавиша F9 (начать запись)
3. **Spotify** → задержка 1 сек → горячая клавиша `ctrl+shift+p` (включить приватный режим)

Все приложения запустятся параллельно, а действия будут выполнены автоматически!

## Структура проекта

```
avto_open/
├── src/
│   ├── main.py           # Точка входа в приложение
│   ├── core/             # Бизнес-логика (Action, Assembly, Executor, Automation)
│   ├── ui/               # Графический интерфейс (PySide6)
│   └── storage/          # Работа с файлами конфигурации
├── requirements.txt      # Список зависимостей Python
└── CLAUDE.md             # Документация для Claude Code
```

## Зависимости

- **Python 3.11+** - язык программирования
- **PySide6** - библиотека для создания графического интерфейса (Qt)
- **PyAutoGUI** - библиотека для автоматизации действий (клики, ввод, горячие клавиши)

## Конфигурация

Сборки сохраняются автоматически в формате JSON:
- **Linux**: `~/.config/avto_open/assemblies.json`
- **Windows**: `%APPDATA%/avto_open/assemblies.json`
